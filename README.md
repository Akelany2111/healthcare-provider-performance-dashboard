# healthcare-provider-performance-dashboard
I recently completed a Healthcare Provider Performance & Revenue Dashboard project that reflects how I like to solve business problems end to end.

I started with Python to generate realistic healthcare claims data and structure the datasets across claims, providers, and insurance tables. This helped simulate a scalable reporting environment and define the data fields needed for meaningful analysis.

Next, I used SQL to clean, join, validate, and aggregate the data. I built summary tables to analyze:

provider performance
payer mix
claims volume
billed vs. paid amounts
denial rates
month-over-month and year-over-year trends

Finally, I used Excel / Google Sheets to turn the analysis into an interactive dashboard with KPI cards, pivot tables, charts, and business insights that make the data easier to understand and act on.

This project highlights the way I enjoy working:
Python for data generation and structure → SQL for analysis and transformation → Excel/Google Sheets for storytelling and decision support

Key metrics included:

Total Claims
Total Billed Amount
Total Paid Amount
Average Paid per Claim
Denial Rate
Provider and Payer Performance Trends

Tools used: Python, MySQL, Excel, Google Sheets, Pivot Tables, KPI Reporting, Dashboard Design

I’m continuing to build projects that showcase both my data analytics and business problem-solving skills.
